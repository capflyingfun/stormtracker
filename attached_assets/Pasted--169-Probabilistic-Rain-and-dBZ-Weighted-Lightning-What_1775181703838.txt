#169 - Probabilistic Rain and dBZ-Weighted Lightning

What & Why

Currently rain rendering uses a hard dBZ cutoff and lightning flashes all eligible cells equally. In reality, lighter storms don’t always produce visible rain and heavier storms produce far more lightning. This task adds probability-based rain and dBZ-weighted lightning selection, creating a more natural and varied look across the storm field.

Done looks like

- Rain appears under clouds based on a probability curve tied to dBZ
- Heavy cells (>=50 dBZ) always show rain
- Light cells (20 dBZ) only show rain ~10% of the time
- Below 20 dBZ, no rain
- Rain presence re-rolls periodically so light-rain cells flicker in and out naturally
- Visually, the storm field has varied rain coverage instead of uniform rain on every cell above a threshold
- Lightning flashes are weighted by dBZ — heavier cells flash far more often than lighter ones

Out of scope

- Changing rain particle appearance, speed, or count per cell
- Changing cloud geometry or glow (handled in #167)
- Rain sound effects
- Lightning timer consolidation (handled in #167 — this task only adds the weighting logic)

Formula

```
rainChance = (dBZ - 20) / 30 * 0.9 + 0.1
```

Clamped: below 20 dBZ = 0%, above 50 dBZ = 100%.

Reference values:
20 dBZ → 10%
25 dBZ → 25%
30 dBZ → 40%
35 dBZ → 55%
40 dBZ → 70%
45 dBZ → 85%
50 dBZ → 100%

Tasks

1. Add rain probability function — Create a function that takes a dBZ value and returns a rain probability using the formula above. Clamp to 0 below 20 dBZ and 1.0 at 50+ dBZ.
1. Roll rain chance per cell on creation/update — When a storm cell is created or its dBZ value updates, roll Math.random() against the rain probability to decide whether that cell displays rain particles. Store the result (showRain boolean) on the cell object.
1. Re-roll rain periodically — Every 30-60 seconds, re-roll the rain chance for cells in the 20-49 dBZ range. Cells at 50+ dBZ always keep rain (skip the roll). This creates natural flickering where light-rain cells occasionally start or stop showing rain. Stagger the re-rolls across cells (don’t re-roll all 67 at once) to avoid visual popping.
1. Wire rain visibility to the roll result — In the rain particle rendering logic, check the cell’s showRain flag. If false, set the rain particle system to visible = false. If true, render rain normally. This integrates with #167’s existing rain throttling and LOD culling — a cell must pass both the probability roll AND the distance/performance checks to show rain.

Relevant files

- docs/js/view3d.js (rain particle creation and animation sections)
- Whatever cell data structure stores per-zone state

Notes

- The re-roll interval (30-60s) should be configurable or at least easy to tune.
- Stagger re-rolls by giving each cell a random offset so they don’t all flip at the same frame.
- This should reduce total rain particle systems rendered at any given time, which is a nice secondary performance win on top of #167’s optimizations.
