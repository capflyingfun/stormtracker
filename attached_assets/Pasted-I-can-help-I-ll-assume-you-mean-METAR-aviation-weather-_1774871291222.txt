Quick checklist of common causes
- Truncated input: the API sometimes returns partial METAR (e.g., "AUTO" only) — verify you receive the full raw METAR string each time.
- Parsing order issues: if your parser expects tokens in a strict order and the METAR uses optional tokens (e.g., RMK, SPECI, TEMPO, PROB, or recent weather), you may skip later tokens.
- Regex greediness: overly broad regex can swallow following tokens (e.g., matching the whole remainder).
- Null/undefined handling: converting absent fields to empty strings vs deleting keys can cause downstream code to skip display.
- Units mismatch: numeric 2-digit values (e.g., visibility "1 1/2SM" vs "9999") may not match an expected pattern and get ignored.
- Locale/encoding: non-ASCII characters or leading/trailing whitespace breaking pattern matches.
- Branching logic: cascaded if/else where one branch returns early and prevents populating other fields.
- API variations: some providers return METAR in separate fields (raw_text, sky_condition[], visibility, wind_dir_degrees) — mapping must cover both shapes.
- Asynchronous race: displaying before parser finishes (e.g., Promise not awaited).

Robust conversion approach (rules + example)
1) Always use the raw METAR string as source-of-truth. If API JSON is present, prefer raw_text/rawMETAR field first.
2) Tokenize conservatively: split on spaces but preserve grouped items (e.g., RMK text). Process tokens sequentially, recognizing optional tokens.
3) Use a defined finite-state parser (not single regex) that checks tokens in typical METAR order and handles optional elements:
   - station (e.g., KSEA)
   - datetime (DDHHMMZ)
   - AUTO/SPECI
   - wind (dddffGggKT or VRB)
   - visibility (e.g., 10SM or 9999)
   - runway visibility (Rxx/...)
   - present weather groups (e.g., -RA, +TSRA)
   - sky conditions (FEW/SCT/BKN/OVC with base)
   - temperature/dewpoint (MM/MM)
   - altimeter (A2992 or Q1013)
   - remarks (RMK …) — everything after RMK should be treated as remarks
4) Defensive parsing: for each token, try multiple patterns and on no match move on but keep the token in a "unparsed" log (so you can inspect dropped pieces).
5) When producing plain language, always explicitly include "Unknown" or "Not reported" for missing fields instead of omitting them — that helps spot parsing gaps.

function parseMetar(raw) {
  const tokens = raw.trim().split(/\s+/);
  let i = 0;
  const out = {station:null, time:null, auto:false, wind:null, visibility:null, wx:[], sky:[], temp_dew:null, altimeter:null, remarks:null, unparsed:[]};

  out.station = tokens[i++]; // assume first
  out.time = tokens[i++];    // assume second (validate with regex)
  while (i < tokens.length) {
    const t = tokens[i];
    if (/^(AUTO|SPECI)$/.test(t)) { out.auto = true; i++; continue; }
    if (/^(VRB|\d{3})(\d{2,3})(G\d{2,3})?KT$/.test(t)) { out.wind = t; i++; continue; }
    if (/^(\d{1,2}SM|9999|\d{4})$/.test(t)) { out.visibility = t; i++; continue; }
    if (/^(R\d{2}[LCR]?\/)/.test(t)) { out.runway = t; i++; continue; }
    if (/^(-|\+|VC)?(RA|SN|DZ|TS|SH|FG|BR|HZ|UP|SQ|PL|GR|GS|IC)+/.test(t)) { out.wx.push(t); i++; continue; }
    if (/^(FEW|SCT|BKN|OVC|VV)\d{3}/.test(t)) { out.sky.push(t); i++; continue; }
    if (/^[M]?\d{2}\/[M]?\d{2}$/.test(t)) { out.temp_dew = t; i++; continue; }
    if (/^(A\d{4}|Q\d{4})$/.test(t)) { out.altimeter = t; i++; continue; }
    if (t === 'RMK') { out.remarks = tokens.slice(i+1).join(' '); break; }
    out.unparsed.push(t); i++;
  }
  return out;
}
